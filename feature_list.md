# Debunk AI (Fact-Checker Pro) - 前后端分离开发计划表

## 阶段一：基础架构与静态页面搭建 (Foundation & Static UI)
**目标：完成项目初始化，搭建基础框架，实现无接口的静态页面和数据库设计。**

### Frontend (前端)
- [ ] **项目初始化**：搭建 React/Next.js 框架，配置 TailwindCSS 等基础依赖。
- [ ] **静态 UI 开发 (学生端)**：对照 prototype.html，实现带进度条的头部、排版良好的阅读区、以及右侧的侧边栏基础结构。
- [ ] **静态 UI 开发 (教师端)**：搭建教师后台的基础页面（文本上传表单、学生列表占位符）。
- [ ] **路由与状态管理**：配置前端基础路由（登录权限拦截、学生大厅、教师后台）。

### Backend (后端)
- [ ] **项目初始化**：搭建 Node.js 或 FastAPI 服务，配置代码结构与数据库连接。
- [ ] **数据库架构设计**：设计 `Users` (教师/学生), `Materials` (原文), `FlawedTexts` (AI生成错误文本), `Annotations` (学生批注数据), `Scores` 表的 Schema。
- [ ] **基础鉴权 API**：实现基于 JWT 的用户注册、登录和角色权限隔离接口。
- [ ] **基础 CRUD API**：提供文章获取、文章上传的基础接口（此时暂时不接入大模型生成逻辑）。

## 阶段二：核心业务与 AI 逻辑接入 (Core Logic & AI Prompts)
**目标：跑通“生成错误文章 -> 学生高亮批注 -> AI打分”的核心完整数据流。**

### Frontend (前端)
- [ ] **核心组件交互 (高亮引擎)**：实现文本点选、区间高亮（Highlight）的复杂交互逻辑，准确记录选中文本在原文中的索引或段落坐标。
- [ ] **批注表单逻辑**：完善右侧弹窗，收集“错误分类（逻辑/事实/幻觉）”与“纠错理由”，组装成标准 JSON 格式。
- [ ] **API 联调 (主链路)**：调用后端接口获取自带漏洞的文章源数据，并将学生的 Highlight 与批注结果提交给后端验证。

### Backend (后端)
- [ ] **AI 刺客生成器 (Prompt 1)**：开发大模型 Prompt 链路，接收知识点或材料，利用 LLM API 生成具有设定“错误密度”的文本数据。
- [ ] **AI 阅卷系统 (Prompt 2)**：开发第二套 Prompt 逻辑，接收学生的批注 JSON，对比标准模型或原文，给出“事实核查评价”与“得分”。
- [ ] **核心业务 API**：编写 `POST /api/generate-flawed-text`（生成题目）和 `POST /api/submit-annotation`（提交答案并触发AI打分）接口。

## 阶段三：控制面板与学情分析 (Dashboard & Analytics)
**目标：完善教师端业务，提供数据可视化与教学难度控制功能。**

### Frontend (前端)
- [ ] **教师端配置面板**：对接接口，实现动态滑块或选项，用于调节AI生成文本的“错误密度”和难度。
- [ ] **学情看板数据绑定**：将统计图表/列表组件对接后端接口，展示班级整体纠错准确率、常犯的思维盲区和高频错误点。
- [ ] **学生端反馈呈现**：提交批注并获得结果后，实时展示 AI 裁判的打分、评语及正确知识点解析弹窗。

### Backend (后端)
- [ ] **统计与聚合 API**：开发 `GET /api/analytics/...` 等数据报表接口，按班级/学生维度提供结构化的汇总统计数据。
- [ ] **难度控制逻辑**：在生成器 Prompt 中支持参数化控制，例如难度1（明显拼写/时间错误）、难度3（隐蔽因果倒置或纯粹AI幻觉）。

## 阶段四：联调优化与部署 (Integration & Deployment)
**目标：消除 Bug，优化提示工程，提升产品体验并完成交付。**

### Frontend (前端)
- [ ] **UI/UX 打磨**：增强交互体验，补充网络请求时的加载状态 (Loading/Skeleton)、平滑过渡动画及错误提示 Toast。
- [ ] **消除“AI味”**：从前端文案、引导流层面调整，使其更贴近真实的教育场景工具（而非单纯套壳 ChatGPT）。

### Backend (后端)
- [ ] **性能与错误兜底**：处理大模型 API 响应超时、生成内容未能严格转为 JSON 格式等异常，编写兜底逻辑。
- [ ] **部署准备**：配置跨域(CORS)、API 访问限流，完成前后端项目和数据库的云端部署。
